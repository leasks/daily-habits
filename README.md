# Daily Habits

An AI-powered personal coaching bot delivered via Telegram. Users send their daily goals and end-of-day reflections; the bot responds with personalised coaching generated by OpenAI and builds a durable memory of each user's patterns over time.

## What it does

1. **Morning check-in** – Users send 3–5 bullet goals for the day along with their most important outcome, any constraints, and blockers. The bot parses the message, stores it, and replies with a personalised coaching response generated by GPT-4.
2. **Midday update** – Users can send a quick progress update; the bot acknowledges it and updates the stored check-in.
3. **End-of-day reflection** – Users report what they achieved, their wins, challenges, and learnings. The reflection is saved and used as context in future coaching sessions.
4. **Daily pings** – A scheduled job (`daily_ping.py`) sends morning, midday, and end-of-day prompt messages to all registered users.
5. **Weekly review** – A scheduled job (`weekly_review.py`) uses the last seven days of check-ins and reflections to extract patterns and stores them as durable memories, giving the coach richer long-term context.

## Tech stack

| Layer | Technology |
|---|---|
| API | Python 3 · FastAPI · Uvicorn |
| Database | PostgreSQL · psycopg |
| AI | OpenAI API (GPT-4.1-mini) |
| Messaging | Telegram Bot API · httpx |
| Config | python-dotenv |

## Project layout

```
app/
  main.py          – FastAPI webhook handler (Telegram → coaching pipeline)
  db.py            – PostgreSQL connection helper
  telegram.py      – Telegram API client (extract updates, send messages)
  parsing.py       – Parse check-in and reflection free-text into structured data
  coaching.py      – Build OpenAI prompt and call the chat completions API
  daily_ping.py    – CLI script: send scheduled prompts to all users
  weekly_review.py – CLI script: run weekly LLM review and store memories
sql/
  schema.sql       – Full database schema (run once on a fresh database)
  migrations/      – Incremental SQL migrations numbered sequentially
```

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | ✅ | PostgreSQL connection string, e.g. `postgresql://app:pass@host/dbname` |
| `TELEGRAM_BOT_TOKEN` | ✅ | Token from [@BotFather](https://t.me/BotFather) |
| `OPENAI_API_KEY` | ✅ | OpenAI API key |
| `JOB_SECRET` | ✅ | Shared secret for authenticating scheduled-job webhook calls |
| `TEST_MODE` | ❌ | Set to `1` / `true` / `yes` to use in-memory stubs instead of real DB / APIs |

## Database setup

### Fresh installation

1. Create a PostgreSQL database and a dedicated `app` role:

   ```sql
   CREATE ROLE app LOGIN PASSWORD 'changeme';
   CREATE DATABASE daily_habits OWNER app;
   ```

2. Run the schema (which includes Row Level Security):

   ```bash
   psql $DATABASE_URL -f sql/schema.sql
   ```

3. Set `DATABASE_URL` to use the `app` role, e.g.:

   ```
   DATABASE_URL=postgresql://app:changeme@localhost/daily_habits
   ```

### Existing installation – run migrations in order

```bash
psql $DATABASE_URL -f sql/migrations/001_add_pending_reply_type.sql
psql $DATABASE_URL -f sql/migrations/002_add_telegram_from_id.sql
psql $DATABASE_URL -f sql/migrations/003_enable_row_level_security.sql
```

## Running locally

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # fill in your values
uvicorn app.main:app --reload
```

Register your bot's webhook with Telegram (replace the placeholder URL):

```bash
curl "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/setWebhook?url=https://your-host/webhooks/telegram"
```

## Scheduled jobs

Both scripts are run as CLI commands (e.g. via cron or a platform scheduler):

```bash
# Send morning prompts
python -m app.daily_ping morning

# Run the weekly review (typically Sunday evening)
python -m app.weekly_review
```
